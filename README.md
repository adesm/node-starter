# Node.js Express Clean Architecture Boilerplate

A simple and robust Node.js boilerplate following clean architecture principles. It's built with TypeScript, Express, PostgreSQL, Prisma (ORM), and Zod for validation.

## Features

- **Module-based Clean Architecture**: Separation of concerns using Routes, Controllers, Services, and Repositories.
- **ORM Integration**: Uses Prisma, the most popular type-safe ORM for Node.js, to interact with PostgreSQL.
- **Authentication**: JWT & bcryptjs pre-configured for user login.
- **Validation**: Strict schema validation using Zod.
- **Testing**: Jest pre-configured for unit testing.
- **Docker Ready**: Multi-stage Dockerfile for optimized production deployments.

## Prerequisites

- Node.js (v18+)
- PostgreSQL Database
- npm or yarn

## Getting Started

1. **Install Dependencies**
   ```bash
   npm install
   ```

2. **Environment Variables**
   Create a `.env` file in the root directory (one is generated by default from Prisma). Update the `DATABASE_URL` to point to your PostgreSQL database.
   ```bash
   DATABASE_URL="postgresql://USERNAME:PASSWORD@localhost:5432/YOUR_DATABASE_NAME?schema=public"
   ```
   *(Optional)* Add a JWT Secret:
   ```bash
   JWT_SECRET="your-super-secret-key"
   ```

## Database Migration & Schema

This project uses Prisma as the ORM. Here is how you manage your database safely.

1. **Schema URL**
   Make sure `DATABASE_URL` is set in your environment before running any migration commands.

2. **Generate Prisma Client**
   After changing the schema, generate the updated TypeScript client:
   ```bash
   npm run prisma:generate
   ```

3. **Create a Migration Locally**
   Use this during development or staging to generate migration SQL and update the local database:
   ```bash
   npm run prisma:migrate -- --name init
   ```

4. **Review the Migration SQL**
   Always inspect the generated migration file under `prisma/migrations/` before committing.

5. **Deploy Migrations in Production**
   Do not use `migrate dev` in production. Use:
   ```bash
   npm run prisma:deploy
   ```

6. **Check Migration Status**
   Use this to verify the deployment state:
   ```bash
   npm run prisma:status
   ```

7. **Prisma naming conventions**
   Use `@@map(...)` and `@map(...)` directly in your Prisma schema to keep database table and column names in snake_case plural form.

8. **Best practice for production migrations**
   - Backup your database before applying migrations.
   - Test migrations first in staging.
   - Keep migration files in version control.
   - Avoid `npx prisma db push` in production; prefer `npm run prisma:deploy`.

## Running the Application

- **Development Mode** (Hot-reloading with `ts-node-dev`):
  ```bash
  npm run dev
  ```
- **Production Mode** (Builds the TS code and runs the JS output):
  ```bash
  npm run build
  npm start
  ```

## Running Tests

To run the unit tests:
```bash
npm run test
```

## Docker Deployment

To build and run the application as a Docker container:
```bash
docker build -t express-boilerplate .
docker run -p 3000:3000 --env-file .env express-boilerplate
```
